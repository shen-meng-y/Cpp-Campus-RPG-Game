# Campus RPG：校园成长冒险游戏

> 一款基于 **C++、Qt 6 和 CMake** 开发的校园主题角色扮演游戏。
> 玩家可以创建角色，在校园中完成任务、探索地点、挑战怪物、购买物品、管理背包，并通过升级解锁新的战斗技能。

---

## 一、项目简介

Campus RPG 是一个以大学校园生活为背景的桌面角色扮演游戏，也是程序设计课程设计项目。

游戏将校园中的学习、运动、作业和考试等元素转化为角色成长与战斗内容。玩家需要管理角色的生命值、经验值、金币和物品，并通过探索、战斗、任务和小游戏不断提升角色能力。

项目目前已完成角色系统、主界面、背包、商店、战斗、急救中心等主要模块，并持续优化任务系统、地图交互、存档稳定性和界面表现。

---

## 二、主要功能

### 1. 角色系统

* 创建并初始化游戏角色
* 显示角色名称、等级、生命值、经验值、金币和攻击力
* 根据经验值完成角色升级
* 支持初级、中级和终极成长形态
* 角色进化后自动弹窗说明新能力
* 不同成长阶段解锁不同战斗技能

### 2. 主界面

* 集中显示角色当前状态
* HP 生命条根据剩余生命值显示不同状态
* EXP 经验条采用蓝色进度显示
* 状态条支持平滑变化动画
* 显示游戏运行时间
* 通过功能按钮进入背包、商店、任务、战斗和急救中心
* 游戏日志区域反馈操作结果和角色变化

### 3. 战斗系统

* 通过校园地图进入战斗探索界面
* 在不同地点探索并发现校园怪物
* 进入独立战斗场景
* 显示玩家和怪物的生命值
* 支持普通攻击、技能攻击和逃跑
* 根据角色成长阶段逐步解锁技能
* 战斗胜利后获得经验值、金币等奖励
* 支持普通怪物与 Boss 战斗

目前设计的校园怪物包括：

| 怪物       | 类型   | 说明            |
| -------- | ---- | ------------- |
| 800 米小怪  | 普通怪物 | 以体测项目为主题的校园怪物 |
| 大作业幽灵    | 中级怪物 | 象征课程大作业压力     |
| 期末周 Boss | Boss | 游戏中的高强度挑战目标   |

### 4. 背包系统

* 查看角色当前拥有的物品
* 显示物品名称、数量和功能
* 使用恢复类或增益类物品
* 使用物品后同步更新角色状态
* 通过可视化界面管理背包内容

### 5. 商店系统

* 使用商品卡片展示商品
* 显示商品名称、价格和说明
* 支持选择商品数量
* 支持加入购物车
* 在购物车中查看商品和总价
* 根据角色金币判断是否可以购买
* 购买完成后更新金币和背包数据

### 6. 急救中心

* 检查角色当前生命状态
* 提供医生诊断信息
* 根据角色受伤情况计算治疗费用
* 支持治疗并恢复生命值
* 加入急救小游戏
* 完成小游戏后可减少部分治疗费用

### 7. 任务系统

任务系统目前仍在持续完善，目标流程包括：

1. 查看任务
2. 接受任务
3. 完成任务目标
4. 领取任务奖励

任务内容可分为日常任务和成就任务，例如游戏时长、击败怪物、角色升级、首次创建角色和首次战斗胜利等。

### 8. 存档与数据管理

* 使用 SQLite 保存游戏数据
* 保存角色基础属性和成长状态
* 为背包、任务和游戏进度持久化提供支持
* 持续优化不同游戏状态之间的数据同步

### 9. 地图与寻路

* 校园地图作为探索与战斗入口
* 已接入 A* 寻路相关逻辑
* 正在根据当前非传统网格地图结构优化路径计算
* 持续改进地图碰撞、位置判断和图片渲染

## 六、开发环境

推荐开发环境：

* Windows 10 或 Windows 11
* Qt 6.11.1
* Qt Creator
* MinGW 64-bit
* CMake
* Git

其他 Qt 6 版本也可能正常运行，但建议保持编译器、Qt 套件和 CMake 配置一致。

---

## 七、运行方法

### 方法一：使用 Qt Creator

1. 下载或克隆项目。

```bash
git clone https://github.com/shen-meng-y/Cpp-Campus-RPG-Game.git
```

2. 使用 Qt Creator 打开项目目录中的 `CMakeLists.txt`。
3. 选择 Qt 6 MinGW 64-bit 构建套件。
4. 执行 CMake 配置。
5. 点击“构建”。
6. 点击“运行”。

### 方法二：使用命令行构建

在已经正确配置 Qt、CMake 和 MinGW 环境变量的情况下执行：

```bash
cmake -S . -B build -G "MinGW Makefiles"
cmake --build build
```

构建完成后，在 `build` 目录中运行生成的程序。

## 八、图片资源说明

本项目当前采用本地文件路径加载图片，**没有使用 Qt Resource System（qrc）**。

运行项目前请注意：

* 不要随意删除或重命名图片目录
* 保持图片文件名与代码中的路径一致
* 建议直接通过 Qt Creator 启动程序
* 移动可执行文件时，需要同时处理对应图片资源路径
* 新增图片后，应同步检查加载路径和大小缩放方式

## 十、已知问题

* 不同操作系统下的键盘输入表现可能不一致
* 某些游戏状态切换后，界面数据可能未及时同步
* 存档读取和覆盖逻辑仍需进一步测试
* 战斗信息显示细节仍需优化
* 当前校园地图并非标准二维数组，A* 寻路需要额外适配
* 部分地图区域可能出现碰撞或位置判断问题
* 不同尺寸图片在窗口中的缩放效果可能不一致
* 本地图片路径可能受到程序工作目录影响
* 多人协作时可能出现文件冲突或分支同步问题
