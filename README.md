### 需求分析与功能拆解
  本项目是一个“<mark>校园RPG冒险游戏系统</mark>”，主要目的是让玩家在游戏中创建自己的角色，并通过完成任务、战斗、购买和使用物品等方式不断提升角色能力。系统需要使用 C++ 进行开发，并体现面向对象程序设计思想，使角色、物品、任务、敌人、商店等内容能够通过不同的类进行管理。

  在功能方面，系统首先需要实现<mark>角色管理</mark>功能。玩家可以创建角色，查看角色的名称、等级、生命值、经验值、金币等基本信息，并且角色的数据能够进行保存和读取，保证关闭程序后再次打开仍然可以继续游戏。其次，系统需要实现背包管理功能，玩家可以获得物品、查看背包、使用物品和删除物品。物品可以分为食物、药品、装备等不同类型，不同物品具有不同的作用。

  系统还需要实现<mark>商店功能</mark>，玩家可以查看商品列表，根据金币数量购买商品，也可以出售背包中的物品，并完成金币结算。任务系统也是游戏的重要部分，玩家可以查看任务、接受任务、完成任务并领取奖励，每个任务都应包含任务名称、任务描述、完成条件和奖励内容。

  <mark>战斗系统</mark>需要支持玩家选择不同敌人进行战斗。战斗过程中，玩家和敌人会相互攻击，生命值会发生变化，系统需要判断战斗胜负，并在胜利后给予玩家经验值和金币奖励。为了让角色有成长过程，系统还需要实现等级成长功能，当经验值达到一定条件后，角色可以升级，并提升生命值、攻击力等属性。

  在<mark>人机交互</mark>方面，系统应提供比较清晰、友好的操作界面，使玩家能够方便地进行***创建角色、查看状态、进入商店、查看任务、战斗和存档***等操作。同时，系统需要对错误输入或不合理操作进行提示，避免程序异常退出。除此之外，本项目还可以结合图形界面或多线程技术，例如使用 Qt 制作游戏界面，或者加入自动存档、游戏计时等功能，使系统更加完整。
